The Story

I've been hosting on Airbnb for years — two separate listings on the Kenyan coast, running quietly in parallel with a tech career. The kind of side-hosting that teaches you a lot about guests, nothing about scale. Then a bigger opportunity arrived that had been building for some time, and it took centre stage.

A boutique property near Mt. Kenya. Barn suites, a café, proper grounds. Not an apartment with a spare room. A real hospitality business that needed real systems — and a tech career's worth of instincts to build them.

"The chaos was real. For an organised mind, watching a hospitality business run on PDFs, manual WhatsApp threads, and memory — with no single source of truth for anything — was genuinely painful."

— The actual motivation.

Eight months in, the property is operational. The café runs. Guests come and go, and come back. Staff get paid. Bills get paid. That's the honest version — and behind it, an enormous amount of work that runs manually, invisibly, and at a scale that makes no sense without a system. A large team could paper over the gaps. Building the right infrastructure feels like the other answer.

But in parallel, something else has been built: a working automation framework. A Playwright + TypeScript test suite. A live availability engine wired to the channel manager. A mobile-first rate card replacing a PDF. An n8n workflow surfacing local demand signals (with more to come). The beginning of a WhatsApp response system using Open Api FAQs. All of it built on top of a business that is simultaneously learning how to exist.

None of this touches the things that genuinely can't be automated yet — great meals, team morale, procurement, general maintenance, the hundred small human decisions that make a stay memorable or forgettable. Those remain entirely human, and probably should. Getting the systems side in order is just the start — but it is a start.

This page is the log of that process. It is not a success story yet. It is a watch this space story — and the space is worth watching.

Why Document This?
Because while large hotel groups can afford expensive software, consultants, and custom systems, small independent hospitality businesses are often left piecing things together with tools that were never really built for them. The question isn't whether this technology exists — it does. The real question is whether it can be made practical, affordable, and adaptable for a smaller operator without losing the human side of hospitality. I'm documenting this as a live experiment: using AI, automation, and systems thinking to build something that works for our independent lodge. It's also a way to lean into the technical side of how I think — solving problems, simplifying operations, and building better systems while running a business.

The Stack

What this looks like from the business side — what each tool does in simple terms.

📅
Channel Manager — Booking Brain
Keeps Airbnb, Booking.com, and direct bookings in sync. One system, one source of truth. Prevents double-bookings automatically.
🌐
Vercel — Guest-Facing Pages
Hosts the rate card, trip planner, and booking form. Fast, mobile-first, no app to download. Lives at rates.idanbarnsuites.com.
⚙️
n8n — Workflow Automation
Runs background workflows: weekly demand digest, alert routing, future WhatsApp triggers. The connective tissue between everything else.
🤖
OpenAI + Claude — AI Layer
OpenAI handles guest FAQ responses. Claude is the architectural partner for decisions, SOPs, and building things correctly the first time.

Mar 18 2026

WhatsApp API investigation — architecture needs rethinking

Migrating to the WhatsApp API would permanently kill the WhatsApp app on the brand number. Staff lose chat history, WhatsApp Web, and human access. Three alternative paths documented. Staged approach recommended.

Tech Decision
Mar 12 2026

False positive alerts fixed — timezone bug in the availability engine

8 false positive double-booking emails overnight. Root cause: JavaScript's new Date() parses date strings as midnight UTC. Kenya is UTC+3 — so midnight UTC is 3am local, and a checkout/checkin on the same calendar day appeared to overlap. Fixed with YYYY-MM-DD string comparison. No false positives since. Smoobu availability check wired into the booking form.

Tech Bug Fix
Mar 10 2026

Smoobu connected — single source of truth is live

Airbnb and Booking.com now syncing through Smoobu. GitHub Actions running every 4 hours with double-booking detection. Key architectural decision: Coral Room held back from Airbnb as an overbooking buffer — only Jade Room listed on Airbnb. Smoobu is now the only place dates get blocked. Never block directly on Airbnb or Booking.com.
